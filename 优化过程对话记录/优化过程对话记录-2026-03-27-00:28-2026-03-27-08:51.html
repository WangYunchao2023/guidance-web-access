🔧 核心函数详解


1. `cortana_execute_flow(cortana_plan)` — **主入口（有经验分支）**

  • **作用**：Cortana 主导的执行入口，接收 JSON 格式执行计划
  • **触发**：有 `user_overrides.yaml` 匹配时，或用户直接传入完整计划
  • **cortana_plan 格式**：
    • 
  {
    "task": "任务描述",
    "search_url": "搜索页URL",
    "search_var": "搜索关键词",
    "filter_criteria": ["过滤条件"],
    "list_urls": ["列表页URL列表"],
    "method": "search_only | navigate_only | both",
    "save_dir": "保存目录"
  }
  • **流程**：解析计划 → 构建探索点 → `explore_with_pagination_v2()` 探索 → 结果过滤 → `final_download()` 下载

    • 2. `cortana_perception_flow(cortana_plan)` — **自感知探索入口（无经验分支）**

  • **作用**：在没有经验规则时，让 Cortana 自主感知页面结构后决定下一步
  • **触发**：`--perception` 参数传入时
  • **流程**：打开 base_url → 自动扫描页面结构（导航链接、搜索框、内容链接）→ 输出感知报告 → 供 Cortana 分析决策

    • 3. `explore_with_pagination_v2(page, intent, exploration_points, translatable)` — **探索与翻页引擎**

  • **作用**：核心搜索遍历函数，负责多 URL 探索 + 翻页
  • **参数**：

    • - exploration_points：dict，格式 {name: {"url": str, "sv": str|None}}

    - sv=None 时由 smart_interact 的 date+primary 逻辑决定填什么

    - sv=str 时用指定字符串填搜索框

  • **核心逻辑**：

    • 1. 遍历每个探索点 URL

    2. 调用 smart_interact() 填搜索框

    3. 调用 get_links_by_text_content_v2() 提取结果

    4. 结果 < 5 条时：降级截短策略（generate_truncated_variants）

    5. 翻页（最多翻到第 5 页，支持多种翻页按钮选择器 + 稳定性检测）


    4. `smart_interact(page, intent, try_date_only, search_var, search_field)` — **界面感知与输入**

  • **作用**：智能感知页面输入框，自动填充搜索词和日期
  • **核心能力**：

    • - 自动识别 keyword、title、start_date 等输入框（通过 id/name/placeholder）

    - layui readonly 日期组件：通过 JS 直接设置值 + 触发 laydate 事件，绕过只读限制

    - 自动点击搜索按钮或按回车


    5. `get_links_by_text_content_v2(page, search_keyword)` — **内容提取器**

  • **作用**：从页面提取关键词匹配的链接列表（v2.9.1 改进版）
  • **核心策略**：**整行提取**

    • - 不依赖"文本节点→向上找链接"的树遍历

    - 先识别页面列表/表格结构（行级元素）

    - 在每行内同时查找文本和链接 → 确保关键词与下载链接在同一条目时正确提取

  • **返回格式**：每个元素含 `{href, text, full_row, date}`

    • 6. `generate_truncated_variants(keyword)` — **关键词截短变体生成**

  • **作用**：当搜索结果过少时，从长到短逐步简化关键词进行降级搜索
  • **策略**：按语义后缀 + 字符长度双重截短

    • 7. `fuzzy_semantic_filter(results, intent)` — **语义过滤**

  • **作用**：日期硬匹配 或 关键词匹配过滤结果
  • **逻辑**：

    • - 有日期时：日期完全一致才保留

    - 无日期时：用 intent['query'] + intent['extra_filter'] 二次过滤


    8. `final_download(page, results, keyword, custom_save_dir)` — **全要素下载**

  • **作用**：遍历结果列表，提取详情页附件并下载
  • **核心能力**：

    • - 语义级文件名智能判定：区分正文文件 vs 辅助文件（起草说明、反馈表），辅助文件必须挂载主标题前缀

    - 增量去重：已存在的文件秒跳过

    - 默认目录：~/Documents/工作/法规指导原则/


    9. `match_keyword_to_selector(search_inputs, keyword)` — **关键词→搜索框智能匹配**

  • **作用**：根据关键词语义（标题词/内容词）选择最合适的搜索框

    • 10. `perception_next_page(page, max_pages)` — **感知模式翻页**

  • **作用**：在自感知流程中翻页，支持多种选择器

    Cookie 预获取 先访问轻量页（robots.txt/favicon.ico）建立会话，避免直接访问被拦截
